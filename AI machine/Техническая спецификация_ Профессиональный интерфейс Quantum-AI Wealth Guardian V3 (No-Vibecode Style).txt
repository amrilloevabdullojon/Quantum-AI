Техническая спецификация: Профессиональный интерфейс и архитектура Quantum-AI Wealth Guardian
Цель документа: Создание детального руководства для разработки MVP, исключающего визуальные и смысловые паттерны "типичных ИИ-проектов". Продукт должен выглядеть как высокотехнологичный терминал для профессиональных участников финансового рынка.
________________
1. Визуальный стиль и Дизайн-система
Основной принцип: Institutional Minimalism. Интерфейс должен напоминать системы уровня Bloomberg Terminal или платформы Goldman Sachs (Marquee), а не яркие лендинги стартапов.
1.1 Цветовая палитра
* Primary: Deep Slate (#0F172A) — фон сайдбара и глубокие тени.
* Surface: Pure White (#FFFFFF) или Light Gray (#F8FAFC) — основной фон рабочих зон.
* Accent: Navy Blue (#1E3A8A) — для активных элементов и кнопок.
* Success: Emerald Green (#059669) — строго для профита и подтвержденных действий.
* Danger/Risk: Crimson Red (#B91C1C) — строго для алертов и зон риска.
* Запрещено: Кислотные цвета, градиенты "Sunset", неоновые свечения.
1.2 Типографика
* Шрифт: "Inter" или "Geist Mono" (для числовых данных).
* Размеры: Использовать сетку 12px / 14px / 16px. Не использовать гигантские заголовки.
* Начертание: Regular и Medium. Избегать Extra Bold.


2. Копирайтинг и Тональность (Анти-AI подход)
Программа должна общаться фактами, а не эмоциями. Текст должен быть сухим и точным.
Концепция
	Запрещенные фразы (Vibe-code)
	Рекомендуемые фразы (Professional)
	Статус системы
	"Магия начинается...", "ИИ подбирает лучшее для вас"
	"Выполняется квантовая оптимизация (QAOA)...", "Расчет матрицы ковариации"
	Рекомендация
	"Вау! Мы нашли потрясающую сделку!", "Попробуйте этот шанс!"
	"Модель предлагает ребалансировку на основе целевой волатильности", "Прогнозная вероятность просадки снижена"
	Ошибки
	"Упс! Что-то пошло не так...", "Нам очень жаль"
	"Ошибка выполнения: Недостаточная глубина исторических данных", "Connection Timeout: AWS Braket"
	

3. Иконки и Графические элементы
* Библиотека: Только Lucide-React или Radix Icons.
* Стиль: Тонкие линии (Stroke 1.5px). Никаких цветных "эмодзи" или 3D-иллюстраций.
* Индикаторы: Вместо мигающих искр ИИ использовать классические скелетоны (skeleton loading) и строгие прогресс-бары.
* Стикеры/Теги: Использовать прямоугольные "Badge" с мягкими цветами: [Quantum-Verified], [Risk: Low], [Sigma-3].


4. Структура экранов (UX Детали)
4.1 Dashboard (Основной терминал)
* Виджеты: Только функциональные. Карточки с четкими границами border: 1px solid #E2E8F0.
* Графики: Использовать Recharts или Lightweight Charts (от TradingView). Тонкие линии, минимум легенд, чистые оси.
4.2 Quantum Insight Log (Журнал вычислений)
Этот раздел должен выглядеть как системный лог терминала, что придает продукту "инженерный" вид.
[SYSTEM] Запуск оптимизации портфеля...
[DATA] Обработка 48 торговых пар через API Binance
[QUANTUM] Подготовка QUBO модели (размерность 128 кубит)
[AWS] Отправка задачи на процессор IonQ Harmony
[INFO] Получено оптимальное решение (Energy: -0.942)
[RESULT] Рекомендовано снижение экспозиции на BTC до 12.4%



5. Технические требования к коду для Claude Code
* Язык: TypeScript с максимально строгой типизацией (Strict Mode).
* Архитектура: Clean Architecture. Разделение на Domain, UseCases, Infrastructure.
* Тестирование: Наличие Unit-тестов для каждого финансового расчета.
* Комментарии: Профессиональные JSDoc комментарии, объясняющие бизнес-логику, а не очевидный код.


6. Финальные штрихи (Polish)
1. Удалить все упоминания "AI" из названий разделов. Вместо "AI Advisor" использовать "Analytics Engine" или "Decision Support".
2. Использовать моноширинный шрифт для всех чисел (цены, проценты). Это выглядит более профессионально и "финансово".
3. Добавить мелкие детали: время последнего обновления данных до миллисекунд, версию билда в футере, статус соединения с облаком AWS зеленым индикатором (dot).
