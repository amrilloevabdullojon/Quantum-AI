Версия: 2.0 (Расширенная)
Статус: Детальная спецификация для разработчиков и инвесторов
Цель документа: Полное описание архитектуры, математической модели, структуры данных и пользовательских сценариев для платформы управления рисками на базе AWS Braket и Amazon Bedrock.


1. Введение и бизнес-логика
Quantum-AI Wealth Guardian решает проблему системного управления рисками для портфелей институциональных и крупных частных криптоинвесторов. Система не занимается высокочастотным трейдингом. Ее основная ценность — Drawdown Mitigation (снижение просадок) посредством математически выверенной ребалансировки.
Продукт продается по модели B2B SaaS (от $2,000/мес для фондов) и B2C Pro (от $199/мес для частных трейдеров).

3. Математическое ядро (Quantum Optimization Engine)
В основе системы лежит квантовый подход к теории Марковица (Modern Portfolio Theory).
3.1 Постановка задачи (QUBO)
Задача минимизации риска переводится в формат Quadratic Unconstrained Binary Optimization (QUBO). Целевая функция минимизирует дисперсию портфеля при заданном ограничении на ожидаемую доходность:
Minimize: f(w) = w^T * Σ * w - λ * (μ^T * w)
Где:
w - вектор весов активов (наш искомый результат)
Σ - матрица ковариации исторических доходностей (до 100x100)
μ - вектор ожидаемых доходностей
λ - коэффициент толерантности к риску (устанавливается клиентом)

3.2 Квантовый алгоритм (QAOA)
Для решения QUBO применяется Quantum Approximate Optimization Algorithm. Классический сервер настраивает параметры схемы (углы beta и gamma), отправляет задачу в AWS Braket, а квантовый процессор (или симулятор) возвращает распределение вероятностей для оптимальных весов.


4. Схема базы данных (Entity-Relationship)
Основа данных хранится в реляционной СУБД PostgreSQL. Основные таблицы:
* users: id, email, password_hash, subscription_plan, api_keys_encrypted, risk_tolerance_score.
* portfolios: id, user_id, total_value, last_synced_at.
* assets: id, symbol (BTC, ETH), current_price, 24h_change.
* portfolio_positions: id, portfolio_id, asset_id, amount, weight_percentage.
* quantum_tasks: id, user_id, status (pending, computing, completed), cost_aws, result_weights_json.
* ai_reports: id, user_id, quantum_task_id, generated_text, recommended_actions_json.


5. Спецификация AI-Агента (Prompt Engineering)
Интеграция с Claude осуществляется через Amazon Bedrock. Для обеспечения стабильности ответов используется жесткий системный промпт.
5.1 Системный промпт (System Prompt)
Ты — Quantum Wealth Advisor, профессиональный риск-менеджер криптофонда.
Твоя задача: интерпретировать результаты квантовой оптимизации портфеля для клиента.

Входные данные (JSON):
- Текущий портфель клиента.
- Текущий риск (Drawdown probability).
- Оптимальный портфель (от квантового алгоритма).
- Снижение риска (дельта).

Правила ответа:
1. НИКОГДА не давай прямых финансовых советов (пиши "Математическая модель рекомендует...", а не "Купи сейчас").
2. Объясняй причины изменения весов (например, "Корреляция между активом X и Y достигла 0.85, квантовый расчет предлагает снизить долю Y").
3. Сохраняй профессиональный, строгий, но эмпатичный тон.
4. Выдай ответ строго в формате:
  - Краткое резюме (Summary)
  - Основные риски (Key Risks)
  - Предлагаемые изменения (Actionable Insights)



6. Детализация интерфейса (UI/UX)
Пользовательский интерфейс должен внушать доверие и демонстрировать "глубину" вычислений.
* Dashboard: Главный экран. Содержит виджет общего баланса, круговую диаграмму распределения активов и спидометр "Risk Score" (например, 72/100).
* Quantum Lab (Квантовая лаборатория): Экран, где пользователь видит прогресс выполнения квантовой оптимизации. Визуализация "энергетического ландшафта" (Heatmap) корреляций.
* AI Advisor Chat: Панель справа. Клиент может задать вопрос: "Почему ты предлагаешь продать Solana?", на что система обращается к логам Bedrock и выдает обоснование.
* Execution (Исполнение): Кнопка "Применить ребалансировку". При нажатии бэкенд через API биржи (Binance) автоматически распродает и покупает нужные активы для приведения портфеля к идеальным весам.


7. Безопасность и AWS Комплаенс
* Изоляция: Все вычисления происходят внутри Amazon VPC. Базы данных находятся в закрытых подсетях (Private Subnets), доступ к ним только через бэкенд.
* Управление ключами: API ключи клиентов от бирж шифруются с помощью AWS KMS (Key Management Service) перед сохранением в БД. Бэкенд расшифровывает их только в момент отправки ордера в оперативную память.
* Мониторинг затрат: Установлены строгие лимиты в AWS Budgets для предотвращения перерасхода кредитов на вызовы AWS Braket.
